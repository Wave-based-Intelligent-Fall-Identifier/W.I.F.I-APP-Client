# W.I.F.Y App — 디자인/설계 기록 (Design Notes)

> 낙상 감지(Fall Detection) **Bluetooth** 모니터링 앱. 디자인 캡처 기반 퍼블리싱 기록 문서.
> 최종 갱신: 2026-06-10

## 1. 기술 방향 (확정)
- **언어/UI**: Kotlin + Jetpack Compose (Material3)
- **폰트**: Wanted Sans (res/font, SIL OFL) — `headline/title` 굵게, 본문 보통
- **기기 통신**: **Bluetooth (BLE)** — 낙상 감지 기기와 직접 통신
- **백엔드**: 원격 서버 없음. **앱 로컬 전용** → Room DB + 로컬 상태 관리
- **아키텍처**: MVVM (ViewModel + StateFlow) + Repository (BLE / Room 분리)
- **네비게이션**: Navigation-Compose, **흐름(flow) 기반 이동** (하단탭 아님)
- **빌드 검증**: Android Studio JBR + gradlew, AGP 9.1.1 / Gradle 9.3.1 / Kotlin 2.2.10 → 베이스라인 BUILD SUCCESSFUL 확인

## 2. 화면 분석 (캡처 8종)

### S1. 등록된 기기 — 블루투스 OFF (스크린샷 121659)
- 타이틀 `등록된 기기`, 우상단 알약 `초기화`(빨강)
- 중앙 안내(회색): "블루투스가 꺼져있어요.\n설정에서 블루투스를 켜 주세요"

### S2. 등록된 기기 — 빈 상태 (스크린샷 122232)
- 중앙 안내: "등록된 기기가 없어요.\n기기를 등록해주세요."
- 외곽선 버튼 `기기 등록하기` → S3(스캔)

### S3. 찾은 기기 — BLE 스캔 결과 (스크린샷 122733)
- 타이틀 `찾은 기기`, 우상단 새로고침(재스캔) 아이콘
- 리스트 행: 신호세기 아이콘(초록/노랑/빨강) + `wify_device_1/2/3` + `연결하기` 버튼(회색 알약)
- `연결하기` → 기기 등록 → S6 그리드에 추가

### S4. 기기 상세 — 선택 모드 (스크린샷 122753)
- 상단 `< wify_device_1` + 우상단 알약 `이름수정`
- 라우터 이미지(안테나 2개)
- `최근 로그`: `⚠ 낙상 감지`(빨강 박스) `3:00` / `⚠ 기기 배터리 부족 알림`(노랑 박스) `3:00`
- 하단 풀폭 검정 버튼 `선택하기` → S7(메인)에서 현재기기로 설정
- `이름수정` → S5

### S5. 기기 이름 수정 모드 (스크린샷 122820)
- 상단 타이틀이 편집 입력 필드(`기기 1`, 회색 하이라이트)로 전환
- 본문 동일, 하단 검정 버튼 `저장하기` → 저장 후 S4 복귀

### S6. 등록된 기기 — 그리드 (스크린샷 122922)
- 상단 `< 등록된 기기` + `초기화`
- 2열 카드: 기기명 + 등록일(예: `2026.06.10`) + 우상단 톱니(설정)
- 상태색: 정상=연회색, **위험=빨강**(`위험 15:40`), **주의=노랑**
- 카드 탭 → S4(상세)

### S7. 메인 모니터링 (스크린샷 111314)
- 좌상단 `현재 기기 : 기기명`
- 우상단 `기기 ☰`(→S6), `기록 ↓≡`(→S8)
- 중앙 `낙상 상태 : 정상`(정상=초록)
- `초기화` 큰 버튼(빨강 테두리, 새로고침), 하단 `사용중`

### S8. 기록(로그) (스크린샷 111358)
- 상단 `< 기록` + 우상단 필터 아이콘 → 드롭다운(`기기 1` 선택/굵게, `기기 2`, `기기 3`)
- 로그 리스트: `⚠ 낙상 감지`(빨강) / `⚠ 기기 배터리 부족 알림`(노랑)

## 3. 네비게이션 플로우 (확정)
```
[등록된 기기] (3상태: BT off S1 / 빈 S2 / 그리드 S6)
   | 빈 상태: "기기 등록하기"
   v
[찾은 기기 스캔 S3] --연결하기--> 기기 등록 --> [그리드 S6]
                                                   | 카드 탭
                                                   v
                                            [기기 상세 S4] --이름수정--> [이름수정 S5] --저장--> S4
                                                   | 선택하기
                                                   v
                                            [메인 모니터링 S7] --기기--> S6
                                                                  --기록--> [기록 S8] (기기별 필터)
```
- 시작점(추정): `등록된 기기`(S1/S2/S6) — 권한/BT/등록 상태에 따라 분기

## 5. 데이터 모델 (Room + BLE, 로컬 전용)
- `Device(id, name, bleAddress, registeredAt, status: NORMAL/WARNING/DANGER, batteryLow, isSelected)`
- `LogEntry(id, deviceId, type: FALL_DETECTED/BATTERY_LOW, message, timestamp)`
- `FallState(deviceId, state: NORMAL/DANGER, inUse, updatedAt)`
- BLE: scan → connect → subscribe(notify) → 낙상/배터리 이벤트 → Room 기록 → UI(StateFlow)
